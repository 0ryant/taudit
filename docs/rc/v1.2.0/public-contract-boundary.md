# v1.2.0-rc.1 Public Contract Boundary Matrix

Source decisions: [ADR 0010](../../adr/0010-public-contract-boundary-and-api-readiness.md), [L2-01/L2-02](code-complete-lanes.md#l2-api-schemas-contracts), and the [contract platform workstream](workstreams/contract-platform.md).

Classification values:

- `public/stable`: public contract already versioned or tested as stable; additive changes only unless a major/breaking path is declared.
- `public/RC`: public contract for the RC, but current-profile or parity evidence must land before stable promotion.
- `internal`: implementation surface; published crate status does not make it the downstream contract.
- `deferred`: not promoted in `v1.2.0-rc.1`.
- `compatibility-only`: accepted for older consumers or aliases, not the current contract to advertise.

## Field Freeze Dependencies

| Dependency | Current evidence | RC decision | Exact evidence needed before freezing |
| --- | --- | --- | --- |
| Output identity | `rule_id` and `fingerprint` are cross-sink proven by [cross-sink tests](../../../crates/taudit-cli/tests/cross_sink_contract.rs) and documented in [ADR 0012](../../adr/0012-public-output-identity-contract.md). `suppression_key` and `finding_group_id` are observed in schemas and sink code, but expanded parity tests are still listed in L5-01. Platform token, source file identity, scan provenance, and event provenance are public/RC until current-profile checks land. | Freeze only `rule_id` and `fingerprint` as already proven. Treat `suppression_key`, `finding_group_id`, platform token, source file identity, scan provenance, and event provenance as pending current-profile dependencies, not stable names solely from this matrix. | L2-04 field definition, L5-01 cross-sink equality across JSON/SARIF/CloudEvents/baseline/suppression lookup, L5-04 CloudEvents mapping, ADR 0017 examples, ADR 0020 conformance. |
| Ordered authority evidence | [ADR 0011](../../adr/0011-ordered-authority-evidence-model.md) names candidate event concepts: `PathMutation`, `SecretMaterialized` or `AuthorityMaterialized`, `HelperExecution`, and `HelperReceivesAuthority`, with `step_index` ordering. Current observed schemas prove exploit projection fields such as `helper_resolution`, `authority_transport`, and `authority_origin`, but not public ordered-event wire names. | Ordered evidence wire names are pending. Do not freeze ADR 0011 event or field names from this matrix. | L2-03 schema/API field definitions, L4 shared event builder, fixtures for positive/reversed/same-step/cross-job/missing-materialization cases, L5 evidence parity projection, and ADR 0020 conformance. |
