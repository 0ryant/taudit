This index normalizes Algol authority-confusion research into taudit rule intake. These are classifier and prioritization rules for customer-safe findings unless an explicit internal feature gate enables witness/disclosure metadata. Corpus hits and source leads are not vulnerability claims.

Evidence levels used here:

- `corpus signal`: public workflow shape exists.
- `source lead`: inspected source suggests helper, cleanup, cache, or output authority handoff.
- `runtime witness`: canary-only proof confirms control and authority reachability.

Statuses:

- `implemented`: present in the current taudit customer-facing rule index, or folded into a present canonical rule.
- `queued`: useful classifier rule not yet present as a canonical customer-facing rule.
- `deferred`: catalog, witness, disclosure, CVE, red-team, or internal-only work that needs a feature gate or separate research lane.
- `suppressed`: duplicate, alias-only, expected behavior, or insufficiently distinct from an implemented classifier.

## Dedupe notes

- The workflow-shell sink list is canonicalized under `gha_workflow_shell_authority_concentration`; individual shell sink names remain match variants, not separate customer-facing rule IDs.
- `gha_tool_installer_then_shell_helper_authority` covers Helm, kubectl, cosign, and similar installer-to-shell flows. Installer-only aliases are suppressed because the Algol handoff explicitly treats installer alone as negative.
- Setup-node, setup-python cache, setup-python pip-install, Docker QEMU, tool-installer, and workflow-shell rules are already present in `docs/rules/index.md`; they are marked `implemented` here even where the backlog still has older implementation-task wording.
- Cross-provider Algol names are kept as queued only when they add a narrower sink or transport than the current customer-facing rule index. Obvious base-shape duplicates are suppressed and mapped to existing rule names in the deferred/suppressed table.
- Runtime-witness rows are still customer-safe classifier intake. Witness artifacts, canary values, source-private details, and disclosure/CVE metadata remain deferred behind an internal feature gate.
