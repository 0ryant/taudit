# taudit-mcp

Generated by McPact.

Source CLI: `taudit`
Trust ceiling: `Reviewed`

## Serve

```bash
cargo run -- serve
```

## Tools

- `taudit_baseline_init` — Snapshot CURRENT findings on the given pipelines into `.taudit/baselines/<pipeline-content-hash>.json`.
- `taudit_baseline_list` — List all waivers across every baseline under `.taudit/baselines/`, sorted by expiry.
- `taudit_baseline_promote` — Append a finding to a baseline as an accepted waiver (taudit's `baseline accept`). Requires --reason >= 10 chars; critical waivers require --severity-override + --expires-at.
- `taudit_baseline_rollback` — Compare pipeline findings against the stored baseline (taudit's `baseline diff`). No write occurs; the result drives any manual rollback decision.
- `taudit_diff` — Diff findings between two pipeline versions.
- `taudit_emit_spec` — Emit a CellOS execution-cell spec that runs taudit scan.
- `taudit_explain` — List built-in rule IDs, or print one rule's full description and remediation.
- `taudit_graph` — Emit graph views (JSON, DOT, Mermaid, or summary) over the authority graph.
- `taudit_invariants_explain` — Explain a built-in invariant by id. Alias for `taudit explain`; taudit-cli has no `invariants explain` subcommand.
- `taudit_invariants_list` — List every loaded invariant (built-in plus any custom).
- `taudit_map` — Show authority map — which steps access which secrets/identities.
- `taudit_remediate_apply` — Apply low-risk remediations with backup + validation + auto-restore. Mutates pipeline files.
- `taudit_remediate_suggest` — Show candidate remediations without modifying files.
- `taudit_scan` — Scan pipeline file(s) for authority findings. Informational; exits 0 unless a structural error occurs.
- `taudit_verify` — Enforce policy invariants; exit non-zero on any violation. Read-only against the workspace.
